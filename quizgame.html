Response 3a:

The program quizzes the user on technological trivia and assigns them a score. The user clicks on the button choice that correctly answers the question which in turn impacts the score. When the user clicks on the correct button, the score increases by 1. When the user clicks on the incorrect choice, the score decreases by 1 and a messagebox appears to notify them of their inaccurate answer.

Response 3b:

The list name is “scorenumbers”. The data contained in the list is the ending score that the user enters which is then saved to “highscores.txt” when they click file, and save. If they choose to open the file, their previous end score will show up in the “enterScorelistbox1”. The listbox is a necessity because everytime the user finishes the quiz, they are able to enter their score and click “save file” in order for them to see what score they need to beat next time they run the program when they click “open file”. If I did not use the list, there would be no way to keep a record of previous scores or add a new one on top of the old score.
